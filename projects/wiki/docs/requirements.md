# KotaServer Wiki 要件定義書

## 1. プロジェクト概要

### 1.1 目的

KotaServer（Minecraft サーバー）のウェブサイト（Astro）に Wiki 機能を追加する。自前の Headless CMS（kotaserver-web-admin）と連携し、CMS で管理するコンテンツを Wiki ページとして静的サイト上で配信する。

### 1.2 現状

| 項目 | 現状 |
|------|------|
| **Website（Astro）** | シングルページのランディングページ。コンテンツはハードコード。CMS 連携なし。 |
| **CMS（kotaserver-web-admin）** | Kotlin/Ktor 製 Headless CMS。ページ管理・メディア管理・ナビゲーション管理・API キー認証付き公開 API を実装済み。 |

### 1.3 ゴール

- CMS で作成・公開したページが、Astro サイト上で Wiki ページとして表示される
- CMS のナビゲーション設定が Wiki のメニューに反映される
- CMS にアップロードした画像が Wiki ページ上で表示される
- CMS でコンテンツを更新したら、Webhook 経由でサイトが再ビルドされる

---

## 2. 機能要件

### 2.1 Wiki ページ表示

- CMS の公開済みページを Wiki ページ一覧として表示できること
- CMS のページを個別の Wiki ページとして表示できること
- ページ本文の Markdown が正しく HTML として表示されること
- ページタイトルがブラウザタブや OGP に反映されること

### 2.2 ナビゲーション

- CMS のナビゲーション設定が Wiki のヘッダーメニューに反映されること
- ナビゲーション項目に紐づくページへのリンクが正しく機能すること
- CMS で設定した並び順が反映されること

### 2.3 メディア表示

- CMS にアップロードした画像が、Wiki ページ本文中で正しく表示されること

### 2.4 自動再ビルド

- CMS でのコンテンツ更新時に、Webhook を通じてサイトが自動的に再ビルドされること

---

## 3. 非機能要件

### 3.1 パフォーマンス

- 静的サイト生成（SSG）により、高速なページ表示を実現する

### 3.2 セキュリティ

- API キーがクライアントサイドに露出しないこと
- API キーがソースコードにコミットされないこと

### 3.3 開発体験

- ローカル環境で CMS のローカルサーバーに接続して開発できること
- 環境変数で接続先を切り替え可能であること

---

## 4. 前提条件

- CMS サーバーが稼働しており、公開 API が利用可能であること
- CMS に API キーが発行済みであること
- CMS に公開済みのページが存在すること

---

## 5. 制約事項

- サイトは SSG（静的サイト生成）で構築する。SSR は使用しない
- 検索機能はスコープ外とする（将来的に Pagefind 等で対応可能）
- ユーザー認証はサイト側では不要（CMS 管理画面のみ）
